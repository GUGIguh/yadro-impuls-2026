# yadro-test-task-spring-2026
Мое тестовое задание в команду по разработки Web-интерфейса для управления базовыми станциями сотовой связи.
Приложение для управления пользователя на **Angular20** и **NG-ZORRO** + **SCSS**

## Демо
Проект разернут на [GitHub Pages](https://gugiguh.github.io/yadro-impuls-2026/) и его можно развернуть локально на своем устройстве 

### Установка и запуск 
- git clone https://github.com/GUGIguh/yadro-impuls-2026.git
- cd yadro-impuls-2026
- npm install
- ng serve
- Проект будет развернут на http://localhost:4200/

## Структура проекта
```
src/app/
├── components/
│   ├── user-list/      # Список пользователей
│   ├── user-detail/    # Детальная карточка пользователя
│   └── user-form/      # Форма создания/редактирования
├── services/
│   ├── user-api.ts     # HTTP-запросы к API
│   ├── user-store.ts   # Локальное состояние
│   └── confirm-delete.ts # модалальное окно для подтверждения удаления
├── models/
│   └── user.model.ts   # Интерфейс пользователя для типов
├── app.component.ts    # Корневой компонент
├── app.config.ts       # Конфигурация приложения
└── app.routes.ts       # Маршруты
```
## Реализуемые функции 
- Удаления пользователя на главной странице и на странице подробной информации
- Редактирование пользователя на главной странице и на странице подробной информации
- Навигация на страницу с допольнительной информации при клике ячейку в таблицу либо на кнопку "Обзор"
- Пагинация при количестве пользователей больше 10
- Создание пользователя
- В таблице выделяются строки последних трех редактированных пользователей 
- Фильтрация таблицы по email и имени пользователя 

## Ключевые решения
**Важно** Для сохранения изменeний был создан отдельный сервис. Так как API только имитирует изменeния, но их нигде не сохраняет, решил сохранять эти изменения локально и отображать таблицу по ним. При обновлении страницы подтягивается версия данных с сервера.

- Использование Signals для реактивных состояний
- Staandalone components без NgModels, все компоненты самостоятельные
- Angular Reactive Forms для формы
